**شرح هذه الدالة المهمة:**

هذه الدالة هي قلب المشروع النابض. لماذا؟ لأنها تحل مشكلة أساسية: النموذج يتوقع 30 ميزة (features) محددة، لكن المستخدم العادي لا يستطيع توفير هذه الـ 30 ميزة لأنه يحتاج إلى معدات طبية متخصصة. لذلك، صممنا هذه الدالة لتقوم بـ "ترجمة" المدخلات البسيطة (8 حقول) إلى 30 ميزة يفهمها النموذج.

**كيف تعمل هذه الدالة؟**

1. **إنشاء مصفوفة فارغة**: ننشئ مصفوفة من 30 صفراً باستخدام `np.zeros(30)`. هذه المصفوفة ستمثل جميع الميزات التي يتوقعها النموذج.

2. **استخراج المدخلات**: نستخرج القيم التي أدخلها المستخدم من القاموس (form_data). نستخدم `form_data.get()` مع قيمة افتراضية في حال كانت القيمة مفقودة، وهذا يمنع حدوث أخطاء.

3. **حساب درجة المخاطر (risk_score)**: هذه فكرة ذكية قمنا بتطويرها. بدلاً من التعامل مع كل ميزة على حدة، قمنا بحساب درجة مخاطر إجمالية من 0 إلى 1 تجمع جميع عوامل الخطر:

   - **العمر**: إذا كان العمر 70+ يضيف 0.25، إذا 60+ يضيف 0.20، وهكذا. لماذا؟ لأن خطر سرطان الثدي يزداد مع التقدم في العمر، خاصة بعد سن 50.
   
   - **مؤشر كتلة الجسم (BMI)**: السمنة عامل خطر معروف، لذلك كلما زاد BMI زادت درجة الخطر.
   
   - **حجم الورم**: الأورام الأكبر حجماً تشير عادة إلى مرحلة أكثر تقدماً من المرض.
   
   - **العقد الليمفاوية**: كلما زاد عدد العقد المصابة، زاد احتمال أن يكون السرطان قد انتشر.
   
   - **الانتشار (Metastasis)**: هذا هو أخطر عامل، وجود انتشار يضيف 0.25 إلى درجة الخطر.
   
   - **التاريخ الطبي**: وجود تاريخ سابق للإصابة يزيد من خطر التكرار.

4. **بناء الميزات**: بعد حساب درجة الخطر، نبدأ في بناء الميزات واحدة تلو الأخرى. على سبيل المثال:

   - `features[0] = 10 + (tumor_size * 0.1) + (risk_score * 4)`: هذه هي ميزة "متوسط نصف القطر" (mean radius). قمنا بتصميمها بحيث تزداد كلما زاد حجم الورم ودرجة الخطر.
   
   - `features[6]`: هذه ميزة "متوسط التقعر" (mean concavity)، وهي من أهم المؤشرات على الأورام الخبيثة. قمنا بتصميمها بحيث تكون أعلى بكثير إذا كان هناك انتشار أو تاريخ مرضي.

5. **حساب القيم المشتقة**: بعد حساب القيم المتوسطة، نحسب قيم الانحراف المعياري (بنسبة 5% إلى 15% من المتوسط) والقيم الأسوأ (بنسبة 1.1 إلى 1.8 ضعف المتوسط). هذا يحاكي بنية مجموعة البيانات الأصلية.

6. **تحديد النطاقات**: نستخدم `np.clip()` للتأكد من أن جميع القيم تقع ضمن النطاقات الواقعية المستمدة من مجموعة البيانات الأصلية. على سبيل المثال، متوسط نصف القطر يجب أن يكون بين 6 و 28.

لماذا صممنا الدالة بهذه الطريقة المعقدة؟ لأننا أردنا أن تكون المدخلات بسيطة قدر الإمكان للمستخدم العادي، مع الحفاظ على دقة النموذج. هذه الدالة تعمل كـ "مترجم" بين لغة المستخدم البسيطة ولغة النموذج المعقدة.

## سابعاً: دوال المصادقة والتحقق

```python
def is_admin():
    if 'user_id' not in session:
        return False
    conn = sqlite3.connect('database/db.sqlite3')
    c = conn.cursor()
    c.execute('SELECT is_admin FROM users WHERE id = ?', (session['user_id'],))
    result = c.fetchone()
    conn.close()
    return result and result[0] == 1

def admin_required(f):
    from functools import wraps
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if 'user_id' not in session:
            flash('Please sign in to access this page.', 'warning')
            return redirect(url_for('login'))
        if not is_admin():
            flash('Access denied. Admin privileges required.', 'danger')
            return redirect(url_for('dashboard'))
        return f(*args, **kwargs)
    return decorated_function
```

**شرح دوال المصادقة:**

هذه الدوال تشكل نظام الأمان في التطبيق. لنفهم كيف تعمل:

**دالة is_admin():**

تقوم بالتحقق من صلاحيات المستخدم الحالي. كيف تعمل؟

1. أولاً، تتحقق من وجود مستخدم مسجل الدخول (عن طريق التحقق من وجود `user_id` في الجلسة).
2. إذا لم يكن هناك مستخدم، تعيد `False` فوراً.
3. إذا كان هناك مستخدم، تتصل بقاعدة البيانات وتسأل: هل هذا المستخدم لديه صلاحيات مدير؟ (is_admin = 1)
4. بناءً على النتيجة، تعيد `True` إذا كان مديراً، أو `False` إذا لم يكن.

**مزخرف admin_required (Decorator):**

هذه تقنية متقدمة في Python تسمح لنا بـ "تزيين" الدوال. ما معنى ذلك؟

بدلاً من كتابة نفس كود التحقق في بداية كل دالة من دوال المدير، نقوم بوضع `@admin_required` فوق الدالة. هذا المزخرف يقوم تلقائياً بما يلي:

1. التحقق من وجود مستخدم مسجل الدخول
2. إذا لم يكن موجوداً، يعيد توجيهه إلى صفحة تسجيل الدخول مع رسالة تحذير
3. إذا كان موجوداً، يتحقق من صلاحيات المدير
4. إذا لم تكن لديه صلاحيات، يعيد توجيهه إلى لوحة التحكم العادية مع رسالة رفض
5. إذا اجتاز جميع الفحوصات، يسمح بتنفيذ الدالة الأصلية

هذا التصميم يتبع مبدأ "لا تكرر نفسك" (DRY - Don't Repeat Yourself)، ويجعل الكود أكثر نظافة وأسهل في الصيانة.

3. **تحويل المدخلات إلى ميزات**: نستدعي دالة `create_full_feature_vector()` لتحويل المدخلات الثمانية إلى 30 ميزة.

4. **توحيد القيم**: نستخدم `scaler.transform()` لتوحيد الميزات بنفس الطريقة التي تم بها تدريب النموذج. هذه خطوة حاسمة لأن النموذج يتوقع بيانات موحدة.

5. **التنبؤ**: نستخدم `model.predict_proba()` للحصول على الاحتمالية. لماذا اخترنا `predict_proba` بدلاً من `predict`؟ لأن `predict` يعطينا فقط التصنيف النهائي (0 أو 1)، بينما `predict_proba` يعطينا الاحتمالية الكاملة (مثلاً 0.85 تعني 85% خطر). هذا يسمح لنا بعرض نسبة الخطورة بدقة.

6. **تصنيف النتيجة**: نحدد ما إذا كانت النتيجة خبيثة (1) إذا كانت الاحتمالية 50% أو أكثر.

7. **تحديد مستوى الخطورة**: نستدعي `get_risk_level()` للحصول على التصنيف واللون والتوصية المناسبة.

8. **الحفظ في قاعدة البيانات**: نخزن جميع المعلومات في جدول predictions، مع ربطها بمعرف المستخدم الحالي.

9. **تخزين النتيجة في الجلسة**: نخزن معلومات التوقع في `session` لأننا سنحتاجها في صفحة النتائج. نستخدم `session.pop()` لاحقاً لاسترجاعها وحذفها.

10. **إعادة التوجيه**: نعيد توجيه المستخدم إلى صفحة النتائج مع رسالة نجاح.

### دالة حذف التوقع

```python
@app.route('/delete-prediction/<int:prediction_id>', methods=['POST'])
def delete_prediction(prediction_id):
    if 'user_id' not in session:
        return jsonify({'success': False, 'message': 'Please sign in first'})
    
    try:
        conn = sqlite3.connect('database/db.sqlite3')
        c = conn.cursor()
        c.execute('SELECT user_id FROM predictions WHERE id = ?', (prediction_id,))
        result = c.fetchone()
        
        if not result:
            return jsonify({'success': False, 'message': 'Record not found'})
        
        if result[0] != session['user_id'] and not session.get('is_admin'):
            return jsonify({'success': False, 'message': 'Unauthorized'})
        
        c.execute('DELETE FROM predictions WHERE id = ?', (prediction_id,))
        conn.commit()
        conn.close()
        
        return jsonify({'success': True})
```

**شرح دالة الحذف:**

هذه الدالة تسمح للمستخدم بحذف أحد توقعاته السابقة. لاحظ كيف نتحقق من الصلاحيات:

1. **التحقق من تسجيل الدخول**: نتأكد من أن المستخدم مسجل الدخول.
2. **التحقق من ملكية السجل**: نتأكد من أن السجل الذي يحاول المستخدم حذفه يعود له بالفعل، أو أن المستخدم مدير (للمدير صلاحية حذف أي سجل).
